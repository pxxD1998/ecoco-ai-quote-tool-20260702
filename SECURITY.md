# Security Policy

## 支援範圍

本 Repo 是純前端求職作品 Demo，不是正式商用報價、會計或稅務系統。安全修正以目前 `main` 分支為主，早期 Git 歷史只保留開發沿革，不另行維護。

## 回報方式

若發現安全問題：

1. 優先使用 GitHub Repository 的 **Security / Report a vulnerability** 私密通報功能（若該功能可用）。
2. 若私密通報功能不可用，可先建立不含漏洞細節、個資、憑證或攻擊內容的 Issue，請 Repository Owner 提供後續私密聯絡方式。
3. 請勿在公開 Issue、PR、截圖、測試資料或 JSON 備份中公開 Token、API Key、真實客戶資料、本機路徑或可直接利用的攻擊內容。

## 資料處理邊界

- App 不需登入，沒有後端，也不會主動上傳報價資料。
- 歷史報價只存於目前瀏覽器的 `localStorage`。
- JSON 匯出檔可能包含使用者輸入的報價與聯絡資料，應由使用者自行安全保管。
- 匯入檔限制 5 MB，並在寫入 localStorage 前驗證 schema、資料型別、金額精度及品項結構。
- 同編號匯入不覆蓋既有報價，而是明確略過。

## 已知非目標

本作品沒有帳號權限、伺服器端驗證、雲端同步、多人協作、電子簽章或正式稅務合規保證。需要上述能力時，應改採經過身分驗證、伺服器端授權、稽核紀錄與正式備份設計的系統。
